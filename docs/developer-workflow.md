# Developer Workflow

Every change follows the same process.

1. Plan
2. Edit
3. Review diff
4. Run

```bash
make check
```

5. Commit
6. Push

## Commit Style

Each commit should represent one logical change.

Examples:

- add package installation library
- use shared package installation library
- add project validation script

## Releases

Meaningful milestones become Git tags and GitHub Releases.
